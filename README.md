# 介绍生成器单页说明文档

## 1. 项目概述

这是一个纯静态的服务器配置介绍 HTML 生成器，主文件为 `index.html`。页面不依赖后端服务、不依赖构建工具，也不需要安装第三方依赖，直接用浏览器打开即可使用。

页面用于快速填写服务器相关配置，并实时生成一段可复制的 HTML 展示代码。生成结果适合粘贴到商品详情页、公告页、落地页、工单回复、销售页面或其他支持 HTML 的编辑器中。

## 2. 运行方式

### 3.1 直接打开

双击 `index.html`，使用浏览器打开即可。

### 3.2 在浏览器中使用

打开页面后，可以直接在左侧填写服务器配置，右侧会实时显示预览效果和生成的 HTML 代码。

### 3.3 无需安装内容

本页面是单文件静态网页，不需要：

- Node.js
- npm
- Vue
- React
- 后端服务
- 数据库
- 打包构建

## 4. 页面主要功能

### 4.1 配置填写

页面左侧提供服务器配置填写区域，目前包含以下类目：

| 图标 | 类目 | 默认值 |
| --- | --- | --- |
| 🧠 | CPU型号 | Intel Xeon E5-2680 v4 |
| 💾 | 内存大小 | 32GB DDR4 |
| 🌐 | 线路 | CN2 GIA / BGP 优化线路 |
| 🔢 | IP数量 | 1个独享 IPv4 |
| ⚡ | 带宽大小 | 100Mbps 独享 |
| 🗄️ | 系统盘大小 | 80GB SSD |
| 💽 | 数据盘大小 | 100GB SSD |
| 🛡️ | 防御 | 100Gbps 防御 |
| 🖥️ | 可用系统 | CentOS、Debian、Ubuntu、Windows Server |

每一项都可以手动修改内容。

### 4.2 显示和隐藏控制

每个配置项右侧都有“显示”开关。

开启时：

- 该项会出现在实时预览中
- 该项会写入复制出来的 HTML 结果中

关闭时：

- 该项不会出现在实时预览中
- 该项不会出现在复制结果中

### 4.3 全部显示和全部隐藏

页面顶部配置区域提供“全部隐藏 / 全部显示”按钮。

作用：

- 当所有配置项都显示时，按钮显示为“全部隐藏”
- 当存在隐藏项时，按钮显示为“全部显示”
- 点击后可以批量切换所有配置项的显示状态

### 4.4 可用系统快捷按钮

“可用系统”类目下方提供快捷按钮：

- CentOS
- Debian
- Ubuntu
- Windows Server

点击按钮可以快速添加或取消对应系统。

按钮高亮表示当前系统已被选中。再次点击高亮按钮，会从“可用系统”内容中移除该系统。

### 4.5 增加其他系统

如果需要添加其他系统，可以在“输入其他系统名称”输入框中填写，例如：

- AlmaLinux
- Rocky Linux
- OpenEuler
- FreeBSD

然后点击“增加系统”按钮，系统名称会自动追加到“可用系统”文本框中。

也可以在输入框中按回车完成添加。

### 4.6 恢复默认示例

点击“恢复默认示例”按钮后，会恢复所有配置项的默认值，并重新打开所有显示开关。

默认内容包括：

- CPU型号：Intel Xeon E5-2680 v4
- 内存大小：32GB DDR4
- 线路：CN2 GIA / BGP 优化线路
- IP数量：1个独享 IPv4
- 带宽大小：100Mbps 独享
- 系统盘大小：80GB SSD
- 数据盘大小：100GB SSD
- 防御：100Gbps 防御
- 可用系统：CentOS、Debian、Ubuntu、Windows Server

### 4.7 实时预览

页面右侧会实时显示最终生成效果。

当左侧进行以下操作时，右侧都会自动更新：

- 修改配置文本
- 显示或隐藏某个类目
- 点击系统快捷按钮
- 添加其他系统
- 恢复默认示例
- 全部显示或全部隐藏

### 4.8 一键复制结果

点击“一键复制结果”按钮后，会复制右侧生成的 HTML 代码。

复制成功后，会显示“已复制到剪贴板”。

复制结果中包含：

- 外层容器样式
- 每个配置项的图标
- 每个配置项的名称
- 每个配置项的值
- 绿色边框和浅色背景
- 自适应样式
- 字体兼容设置

## 5. 生成结果说明

生成结果是一段独立 HTML 片段，不包含完整的 `<!DOCTYPE html>`、`<html>`、`head`、`body` 结构。

它适合嵌入到已有页面中，例如：

```html
<div style="width:100%;max-width:760px;margin:0 auto;...">
  ...
</div>
```

生成内容只保留当前显示状态开启的配置项。

如果某个配置项内容为空，即使显示开关开启，也不会生成该项。

## 6. 图标持久化说明

### 6.1 图标来源

当前图标使用 emoji 字符，例如：

- 🧠
- 💾
- 🌐
- 🔢
- ⚡
- 🗄️
- 💽
- 🛡️
- 🖥️

这些图标写在 JavaScript 的 `fieldConfigs` 配置数组中。

### 6.2 复制后是否保留

会保留。

生成 HTML 时，代码会将图标字符写入结果中。用户点击“一键复制结果”后，复制出来的 HTML 片段会包含这些图标。

### 6.3 字体兼容

为提升不同系统上的图标显示稳定性，页面和生成结果中使用了 emoji 字体兼容栈：

```css
'Segoe UI Emoji','Apple Color Emoji','Noto Color Emoji','Microsoft YaHei',sans-serif
```

常见平台兼容情况：

| 平台 | 主要字体 |
| --- | --- |
| Windows | Segoe UI Emoji |
| macOS / iOS | Apple Color Emoji |
| Linux / Android 部分环境 | Noto Color Emoji |
| 中文环境兜底 | Microsoft YaHei |

## 7. 自适应说明

页面和生成结果都做了响应式适配。

### 7.1 编辑页面适配

编辑页面包含多个断点：

- 大屏：左右双栏布局
- 900px 以下：表单和预览改为上下布局
- 768px 以下：字段改为单列
- 640px 以下：缩小页面间距
- 420px 以下：按钮改为满宽排列

### 7.2 生成结果适配

复制出来的 HTML 结果也包含内联自适应样式，主要包括：

```css
width:100%;
max-width:760px;
font-size:clamp(14px,3.6vw,16px);
overflow-wrap:anywhere;
word-break:break-word;
min-width:0;
```

这些样式可以降低以下问题出现概率：

- 手机端文字溢出
- 长线路名称撑破布局
- IP 或系统名称过长导致横向滚动
- 图标和文字错位
- 不同屏幕字号不协调

## 8. 代码结构说明

整个页面集中在 `index.html` 中，主要分为三部分：

### 8.1 HTML 结构

页面主体结构包括：

- 顶部说明区域 `.hero`
- 左侧配置面板 `.panel`
- 右侧实时预览面板 `.panel`
- 表单容器 `#fields`
- 预览容器 `#preview`
- 生成代码文本框 `#output`

### 8.2 CSS 样式

样式集中在 `<style>` 中。

主要样式模块：

- 页面背景和字体
- 面板卡片样式
- 表单字段布局
- 图标样式
- 显示开关样式
- 快捷系统按钮样式
- 右侧预览样式
- 代码输出框样式
- 响应式媒体查询

### 8.3 JavaScript 逻辑

交互逻辑集中在 `<script>` 中。

主要函数说明：

| 函数 | 作用 |
| --- | --- |
| `escapeHtml(value)` | 转义 HTML，防止用户输入破坏生成结构 |
| `createField(config)` | 根据配置动态创建表单项 |
| `getFieldValue(key)` | 获取某个字段的输入值 |
| `isVisible(key)` | 判断某个字段是否开启显示 |
| `splitSystems(value)` | 将可用系统字符串拆分为数组 |
| `setSystems(items)` | 设置可用系统内容并去重 |
| `updateSystemButtons()` | 根据文本框内容同步快捷按钮高亮状态 |
| `toggleSystem(system)` | 点击快捷按钮时添加或移除系统 |
| `addCustomSystem()` | 添加用户输入的其他系统 |
| `getVisibleSpecs()` | 获取当前需要生成的配置项 |
| `buildProductHtml()` | 构建最终可复制的 HTML 片段 |
| `updateOutput()` | 更新实时预览和代码输出 |
| `copyOutput()` | 复制生成的 HTML 结果 |
| `resetDefaults()` | 恢复默认示例值和显示状态 |
| `toggleAll()` | 全部显示或全部隐藏 |

## 9. 配置项维护说明

所有生成类目都集中在 `fieldConfigs` 数组中。

当前结构示例：

```javascript
{ key: "cpu", label: "CPU型号", icon: "🧠", value: "Intel Xeon E5-2680 v4" }
```

字段含义：

| 字段 | 含义 |
| --- | --- |
| `key` | 字段唯一标识，不能重复 |
| `label` | 页面和生成结果显示的类目名称 |
| `icon` | 类目前面的图标 |
| `value` | 默认示例值 |
| `full` | 是否占满整行 |
| `textarea` | 是否使用多行文本框 |
| `options` | 快捷按钮选项，仅可用系统使用 |

### 9.1 新增普通配置项

如果要新增一个普通类目，可以在 `fieldConfigs` 中添加：

```javascript
{ key: "price", label: "价格", icon: "💰", value: "199元/月" }
```

添加后会自动支持：

- 表单显示
- 显示/隐藏开关
- 实时预览
- 一键复制
- 恢复默认示例
- 全部显示/隐藏

### 9.2 新增多行配置项

如果内容较长，可以添加：

```javascript
{ key: "remark", label: "备注", icon: "📌", value: "支持升级配置", full: true, textarea: true }
```

`full: true` 表示占满整行，`textarea: true` 表示使用多行文本框。

### 9.3 修改默认值

例如要修改默认 CPU，只需要修改：

```javascript
{ key: "cpu", label: "CPU型号", icon: "🧠", value: "Intel Xeon Gold 6133" }
```

### 9.4 修改可用系统快捷按钮

当前可用系统配置：

```javascript
{
  key: "systems",
  label: "可用系统",
  icon: "🖥️",
  value: "CentOS、Debian、Ubuntu、Windows Server",
  full: true,
  textarea: true,
  options: ["CentOS", "Debian", "Ubuntu", "Windows Server"]
}
```

如果要增加快捷按钮，例如增加 `Rocky Linux`：

```javascript
options: ["CentOS", "Debian", "Ubuntu", "Windows Server", "Rocky Linux"]
```

如果希望默认值中也包含它，需要同步修改：

```javascript
value: "CentOS、Debian、Ubuntu、Windows Server、Rocky Linux"
```

## 10. 安全处理说明

页面生成 HTML 时使用了 `escapeHtml(value)` 对用户输入进行转义。

它会处理：

- `&`
- `<`
- `>`
- `"`
- `'`

这样可以避免用户输入中包含 HTML 标签或特殊字符时破坏生成结果结构。

例如用户输入：

```html
<script>alert(1)</script>
```

生成结果会被转义为普通文本显示，而不是作为脚本执行。

## 11. 剪贴板兼容说明

一键复制优先使用：

```javascript
navigator.clipboard.writeText(html)
```

如果浏览器不支持或权限受限，会回退到：

```javascript
outputEl.select();
document.execCommand("copy");
```

这样可以提高在不同浏览器和本地文件打开环境下的复制成功率。

## 13. 使用建议

### 13.1 用于商品页

建议填写清晰、简短、标准化的配置内容，例如：

```text
CPU型号：Intel Xeon E5-2680 v4
内存大小：32GB DDR4
带宽大小：100Mbps 独享
防御：100Gbps 防御
```

### 13.2 用于移动端展示

如果目标页面主要在手机端展示，建议避免过长的连续英文或无空格字符串。

虽然页面已经加入 `overflow-wrap:anywhere`，但更短的文案通常更美观。

### 13.3 用于不同平台粘贴

如果粘贴目标平台支持 HTML，建议复制右侧代码框中的完整 HTML。

如果目标平台不支持 HTML，可能只会显示源代码或过滤样式，这种情况需要使用目标平台支持的编辑方式。

## 14. 常见问题

### 14.1 为什么我关闭某个类目后复制结果没有它？

这是正常行为。关闭显示开关后，该类目不会进入生成结果。

### 14.2 为什么某个类目打开了但没有生成？

如果该类目的输入内容为空，即使显示开关打开，也不会生成该类目。

### 14.3 为什么快捷系统按钮没有高亮？

快捷按钮高亮状态由“可用系统”文本框内容决定。

如果文本框中包含 `CentOS`，CentOS 按钮会高亮。如果手动删除该文字，高亮会取消。

### 14.4 为什么复制后图标在某些地方显示不一样？

emoji 图标的外观由系统和字体决定。Windows、macOS、iOS、Android 上图标样式可能略有差异，但字符本身会保留。

### 14.5 可以把图标换成图片吗？

可以，但当前实现使用 emoji，优点是无需图片资源、复制更方便、单文件即可运行。如果换成图片，需要保证图片地址在目标页面可访问。

## 16. 维护注意事项

1. 修改字段时优先改 `fieldConfigs` 数组。
2. 不要重复使用相同的 `key`。
3. 如果新增字段内容较长，建议加 `full: true, textarea: true`。
4. 如果修改图标，建议使用常见 emoji，以保证显示兼容性。
5. 如果修改生成 HTML 样式，注意保留 `width:100%`、`max-width`、`min-width:0` 和 `overflow-wrap:anywhere`，这些对移动端适配很重要。
6. 如果增加用户输入内容到生成结果，应该继续使用 `escapeHtml()` 转义。
7. 如果调整复制逻辑，建议保留当前回退复制方案，以兼容更多浏览器。

## 17. 总结

该单页是一个轻量、可直接运行的服务器配置介绍生成工具。它通过表单配置、实时预览和一键复制功能，帮助快速生成带图标、绿色 UI、自适应布局的 HTML 配置介绍片段。

它的核心特点是：

- 单文件运行
- 无需依赖
- 支持图标持久化
- 支持字段显示/隐藏
- 支持可用系统快捷选择
- 支持自定义增加系统
- 支持一键复制 HTML
- 支持电脑、平板、手机端自适应
